Release Documentation Gaps

AreaGapNeeded Before v1.0
InstallFinal public release will be a signed .app; current docs describe local bundle assembly.Replace development launch path with final download/install/open/remove/update flow.
PermissionsPlugin permission grants and secrets are documented as contracts, not complete user flows.Add final screenshots and Keychain/permission grant steps once implemented.
Plugin TransportswebView, websocket, and dynamic nativeSwift are contract surfaces with incomplete host execution support.Either finish implementation or mark as preview in release docs.
Schema ValidationCLI validates Swift models, not every JSON Schema constraint.Add schema validation to CI or document Swift validation as canonical for v1.0.
Theme AssetsTheme asset existence and script/CSS bridge policy need hardening.Validate asset paths and document security rules for executable web assets.
ScreenshotsManuals currently use reusable SVG diagrams and prose, not final app screenshots.Add final DK panel, settings window, package install, weather, system monitor, and theme screenshots.
Third-party IntegrationsService setup varies by plugin and is not yet deeply documented.Add plugin-specific pages for Spotify, YouTube, Home Assistant, OBS, OctoPrint, UniFi, Discord, and AI vendors.
Repo Hygiene.DS_Store files are present in source/example/test trees.Remove before release archives and ensure ignore rules prevent recurrence.
PDF QAManuals have not yet been visually printed to PDF in this pass.Open each HTML manual and inspect generated US Letter PDFs.

Recommended Final Documentation Pass

  1. Ship or freeze the final app-bundle install flow.
  2. Capture screenshots from the final panel and settings UI.
  3. Refresh plugin and theme inventory counts from the release branch.
  4. Run ./scripts/validate-release.sh.
  5. Open docs/manuals/index.html and print each manual to PDF.
  6. Review every Gap, Alpha, and Risk callout for release wording.
