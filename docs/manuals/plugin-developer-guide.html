1. Concepts

A functional plugin owns behavior: actions, data, external integrations, applet pages, widgets, or background work. It should not own the visual theme. QuakeKit’s host and active theme decide final presentation.

ConceptMeaning
ActionA command the host can invoke, such as refresh weather, toggle OBS recording, or start a timer.
Data streamA named stream of values a view can display, such as CPU load or current track metadata.
ViewA page, widget, or dual-purpose visual surface backed by plugin data or local web assets.
PermissionA declared trust requirement such as network, filesystem, audio capture, or local process access.

2. Package Layout

A packaged plugin is a directory ending in .quakekitplugin with a required manifest.json.

weather.quakekitplugin/
  manifest.json
  weather.sh
  index.html
  assets/

Installable archives may be .tar, .tar.gz, or .tgz files containing one plugin directory.

4. Settings

Settings are persisted by the host and exported to process plugins as QUAKEKIT_SETTING_*. If a setting declares environment, the host also exports that exact variable name.

FieldUse
typestring, integer, number, boolean, choice, or secret.
groupSettings section name for richer UI.
orderStable sort key within a group.
uiControlPreferred editor: text field, slider, toggle, segmented, popup, file picker, secure field.
restartRequiredWhether the plugin or app should restart before the change takes effect.
 
{
  "id": "location",
  "title": "Location",
  "type": "choice",
  "defaultValue": "Charlotte, NC",
  "choices": ["Charlotte, NC", "Raleigh, NC", "New York, NY"],
  "environment": "QUAKEKIT_WEATHER_LOCATION",
  "group": "Forecast",
  "order": 10,
  "uiControl": "popup",
  "help": "Primary weather location."
}

5. Actions And Data Streams

Actions are invokable commands. Data streams describe values that views can display. Action argument schemas and stream value schemas use the same field shape: id, title, type, required, and defaultValue.

"actions": [
  {
    "id": "timer.start",
    "title": "Start Timer",
    "argumentSchema": [
      { "id": "minutes", "title": "Minutes", "type": "integer", "required": true }
    ]
  }
],
"dataStreams": [
  {
    "id": "timer.state",
    "title": "Timer State",
    "valueSchema": [
      { "id": "remaining", "title": "Remaining Seconds", "type": "integer" }
    ]
  }
]

6. Views And Layouts

Views bridge a plugin into the panel shell. A plugin may expose a full applet page, a compact widget, or both.

FieldValuesPurpose
typenative, webCanvas, webDocument, text, dataDrivenRenderer hint.
presentationpage, widget, pageAndWidgetWhere the host should place the view.
layoutgrid, fullScreen, halfLeading, halfTrailing, halfAndGrid, twoHalves, thirds, quartersPage layout hint.
columnSpan, rowSpanintegerWidget grid sizing.

7. Permissions

Declare the minimum permissions needed. Permissions are part of the host trust contract and should be treated as user-visible security metadata.

"permissions": [
  { "network": { "hosts": ["api.open-meteo.com"] } },
  { "filesystem": { "paths": ["~/Documents/QuakeKit"], "write": false } },
  { "audioCapture": {} }
]

GapPermission prompting and enforcement are not yet complete across all host surfaces. Still declare permissions accurately now so packages are ready for stricter hosts later.

9. Complete Examples

Minimal Shell Widget

{
  "$schema": "../../schemas/functional-plugin.schema.json",
  "id": "hello_status",
  "name": "Hello Status",
  "version": "1.0.0",
  "apiVersion": "0.1",
  "entry": { "transport": "shell", "command": "hello.sh" },
  "capabilities": ["dataProvider", "actionProvider", "viewProvider"],
  "actions": [{ "id": "hello.refresh", "title": "Refresh" }],
  "dataStreams": [{ "id": "hello.status", "title": "Status" }],
  "views": [{
    "id": "hello.widget",
    "title": "Hello",
    "type": "dataDriven",
    "presentation": "widget",
    "dataStreamID": "hello.status",
    "columnSpan": 1,
    "rowSpan": 1
  }]
}
#!/bin/sh
printf '{"title":"Hello","value":"Ready","detail":"QuakeKit plugin online"}\n'

10. Validate And Install

swift run quake-probe --validate-plugin Examples/Plugins/weather.quakekitplugin/manifest.json
swift run quake-probe --run-plugin-action Examples/Plugins/system-monitor.quakekitplugin/manifest.json system.refresh
swift run quake-probe --install-package ./Hello.quakekitplugin
swift run quake-probe --install-package ./Hello.quakekitplugin.tar.gz

Before submitting a plugin to the project, run:

./scripts/validate-release.sh

11. Current Gaps

  • Secret setting storage needs production Keychain handling.
  • Long-lived stream subscription semantics need a deeper host lifecycle contract.
  • Dynamic Swift plugin signing, loading, and sandbox boundaries are not final.
  • Web view bridge APIs should be documented once the host bridge stabilizes.
  • Schema-only constraints should eventually be validated directly in CI.
