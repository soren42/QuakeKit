QuakeKit Platform Overview

QuakeKit is a Swift-native platform for the DK-Quake / ARIS-68 touch display and control surface. The macOS host is the first implementation target; the repo is organized around device adapters, a runtime core, language-neutral plugin contracts, and host apps.

Current API version: 0.1 SwiftPM package macOS 14+ AGPL-3.0-or-later Non-commercial protocol caveat

Hardware Status

Alpha caveat: QuakeKit is not ready as an end-user DK-Suite replacement. It is a hardware/platform bring-up tree for developers and early testers.

Confirmed in the current tree

  • HID control input is decoded.
  • HID touch input is decoded.
  • Knob-ring output can be toggled and themed at runtime.
  • The AppKit panel host renders on the physical DK display.
  • The native shell has Home, Widgets, Apps, Themes, and Runtime pages.
  • Plugin and theme manifests build, decode, and validate against current Swift models.

Still under bring-up

  • Out-of-process plugin execution is not the default runtime path yet.
  • Web and native applet rendering beyond manifest tiles is still a next layer.
  • Permission grant UI, persisted plugin registry, and Keychain-backed secrets are roadmap items.
  • Hardware behavior should still be reported with Mac model, macOS version, display mode, and probe output.

Display and HID Model

The DK-Quake panel is exposed to macOS as a normal external display. QuakeKit does not push framebuffer content through HID. Instead, the macOS host renders a native AppKit window onto the DK screen.

Surface Current model Implementation notes
Display Host-owned AppKit window on an external display. The panel host looks for a Quake-like screen size of 1920x480 or 480x1920, then places a full-frame borderless window unless debug mode is requested.
Control HID Vendor HID interface for control reports, knob events, state responses, safe commands, and knob-ring output. Known control specs include VID/PID 0x4158/0x514B and 0x5012/0x6817 with usage page 0xFF60.
Touch HID Touch reports are decoded into normalized runtime touch points. The supported touch spec is VID/PID 0x0712/0x0010; macOS may also expose the touch collection as a standard digitizer usage page 0x000D, usage 0x0004.
Safe output Wake, keep-alive, firmware query, mic query, luminance query, brightness, and knob-ring lighting. DFU / firmware-download behavior must remain out of probe commands, UI controls, tests, examples, and plugin APIs.

.build/debug/quake-panel --display-test --no-hid

# Normal hardware mode; opens the DK display and starts HID.
.build/debug/quake-panel

Theme Plugin Contract

Theme plugins own presentation, not behavior. A theme package ends in .quakekittheme and contains a required theme.json manifest. Assets are optional and referenced from the manifest.

Examples/Themes/focus-dark.quakekittheme/
  theme.json
  assets/
Area Fields Current expectations
Identity id, name, version, apiVersion, kind apiVersion is currently 0.1; kind must be theme; ids must match ^[a-z0-9][a-z0-9_-]*$.
Palette palette.mode, palette.colors, semanticColors Colors are literal #RRGGBB or #RRGGBBAA. Semantic colors provide host fallback roles for native controls, widgets, and future web bridges.
Typography and metrics typography, metrics Optional font preferences, type scale, corner radius, border width, spacing, and density.
Components tile, tab, statusRow, gauge, chart Component styles may reference semantic colors, palette tokens, or literal colors. The host falls back to semantic colors when component styles are absent.
Hardware presentation hardware.knobRing Themes map semantic ring states to color, intensity, and animation. Themes do not send HID commands or own device state.
Options options Current shell supports persisted overrides for palette.colors.<token>.value, metrics.cornerRadius, metrics.spacing, and metrics.density. 
{
  "id": "focus_dark",
  "apiVersion": "0.1",
  "kind": "theme",
  "palette": {
    "mode": "dark",
    "colors": {
      "accent": { "value": "#7CFFD1", "configurable": true }
    },
    "semanticColors": {
      "accent": "accent"
    }
  },
  "hardware": {
    "knobRing": {
      "enabled": true,
      "focus": { "color": "accent", "intensity": 0.7, "animation": "pulse" }
    }
  }
}

Functional Plugin Contract

Functional plugins own behavior: actions, data streams, applet pages, widgets, adapters, and background work. Packaged plugins end in .quakekitplugin and contain manifest.json. Loose JSON manifests are still supported for early validation.

Examples/Plugins/weather.quakekitplugin/
  manifest.json
  index.html
  assets/
Area Contract
Entrypoint entry.transport is one of stdioJSONRPC, websocket, nativeSwift, webView, shell, or php. Optional fields include command, arguments, and url.
Capabilities Discovery flags: settings, eventPublisher, eventSubscriber, dataProvider, actionProvider, viewProvider, deviceProvider, and backgroundWorker.
Permissions Explicit declarations for sensitive access: network hosts, named secrets, scoped filesystem paths, input synthesis, audio capture, and local process access.
Actions Invokable commands with stable ids, titles, and optional ordered argument schemas.
Data streams Published values with stable ids, titles, and optional value schemas consumed by views, host surfaces, or future plugin bridges.
Views Applet pages and widgets with optional type, presentation, entry path, backing stream, grid spans, and preferred dimensions. 
{
  "id": "weather",
  "apiVersion": "0.1",
  "entry": { "transport": "webView", "url": "index.html" },
  "capabilities": ["dataProvider", "viewProvider"],
  "permissions": [
    { "network": { "hosts": ["api.weather.gov", "api.open-meteo.com"] } }
  ],
  "actions": [
    { "id": "weather.refresh", "title": "Refresh Forecast" }
  ],
  "dataStreams": [
    { "id": "weather.current", "title": "Current Weather" }
  ],
  "views": [
    {
      "id": "weather.canvas",
      "type": "webCanvas",
      "presentation": "pageAndWidget",
      "entryPath": "index.html",
      "dataStreamID": "weather.current"
    }
  ]
}

Knob-Ring Runtime Arbitration

The knob ring is treated as a shared physical indicator. Functional plugins and host subsystems should request semantic states, not raw LED colors. The runtime resolves competing requests, then maps the winning semantic state through the active theme.

Priority Source Typical use
0 idle Theme default when no active request exists.
20 focus Touch, knob movement, selected controls, or active navigation.
40 pluginStatus Normal plugin status such as healthy, loaded, or refreshed.
60 warning Attention states that should not fully interrupt the user.
80 danger Urgent plugin or host conditions.
100 systemCritical Host-critical conditions that should override lower-priority requests.

Resolution rules are deterministic: expired requests are pruned, higher priority wins, and newer requests win ties. If no request is active, the runtime uses the theme's idle state. If the active theme disables the ring or does not define a usable state, the host turns the ring off.

Functional plugin detail surfaces also have an initial data path: data-driven views can run a declared refresh action, store the payload in PluginDataStore, and display the latest snapshot while the longer-lived stream subscription engine is still under construction.

// Current runtime shape.
KnobRingRequest(
  source: "weather",
  state: .warning,
  priority: .warning,
  ttl: 2.5
)

Validation Commands

Build and self-test

swift build
swift run quake-probe --self-test
swift run quake-test

Enumerate HID

swift run quake-probe
.build/debug/quake-probe --all-hid

Validate functional plugins

swift run quake-probe --validate-plugin Examples/Plugins/echo-plugin.json

for f in Examples/Plugins/*.json Examples/Plugins/*.quakekitplugin/manifest.json; do
  swift run quake-probe --validate-plugin "$f"
done

Run plugin actions

swift run quake-probe --run-plugin-action Examples/Plugins/system-monitor.quakekitplugin/manifest.json system.refresh
swift run quake-probe --run-plugin-action Examples/Plugins/echo-plugin.json echo.say

Validate themes

for f in Examples/Themes/*.quakekittheme/theme.json; do
  swift run quake-probe --validate-theme "$f"
done

swift run quake-probe --listen --wake

Near-Term Roadmap

Phase 1: Host runtime

  • Keep hardening AppKit placement on the DK display.
  • Connect runtime event bus paths cleanly to HID touch and knob events.
  • Broaden page model rendering and local action dispatch.
  • Add a Keychain-backed secret store abstraction.

Phase 2: Plugin runtime

  • Persist a plugin registry in Application Support.
  • Add trusted in-process Swift adapters for first-party integrations.
  • Implement out-of-process JSON-RPC over stdio.
  • Add permission grant UI, persisted grants, and data subscriptions.

Phase 3: Control center

  • Embed dashboard pages with WKWebView.
  • Build native settings and editor UI.
  • Move integrations such as Home Assistant, music, metrics, and calendar through the plugin surfaces.
  • Document OS-agnostic protocol behavior for future Windows and Linux hosts.

Tester focus: the most useful reports still include quake-probe --all-hid output, physical display mode and rotation, confirmation of touch/knob/LED behavior, and whether the rendered panel is visible on the DK screen.
